# Security Specification - Market Auto DZ

## 1. Data Invariants
- **User Integrity:** A user cannot set their own `role` or `isVerified` status. These are system/admin-only fields.
- **Ad Ownership:** Only the creator (or admin) can update or delete an advertisement.
- **Immutability:** `createdAt` and `userId` fields cannot be changed after creation.
- **Chat Privacy:** Only participants of a chat can read messages or the chat document itself.
- **Relational Integrity:** Messages must belong to a valid chat.
- **Admin Supremacy:** The account `dalinadjib1990@gmail.com` is the Master Admin and bypasses standard ownership checks.

## 2. The "Dirty Dozen" Payloads (Denial Tests)
1. **The Privilege Escalation:** User tries to create/update their profile with `"role": "admin"`.
2. **The Verification Hack:** User tries to set `"isVerified": true` on their profile or ad.
3. **The Identity Theft:** User tries to create an ad with a `userId` that is not their own.
4. **The Shadow Edit:** User tries to update an ad belonging to another user.
5. **The Chat Spy:** User tries to read a chat document where they are not a participant.
6. **The Message injection:** User tries to post a message into a chat they aren't part of.
7. **The Time Travel:** User tries to manually set a `createdAt` date in the past.
8. **The Ghost Deletion:** User tries to delete another user's ad.
9. **The Oversized Poisoning:** User tries to send a 1MB string for a `title` field.
10. **The ID Poisoning:** User tries to use a non-alphanumeric string as a document ID.
11. **The Orphaned Message:** User tries to create a message for a non-existent `chatId`.
12. **The Terminal State Unlock:** User tries to "un-sell" a car by changing status from `sold` back to `active` (if terminal locking is applied).
